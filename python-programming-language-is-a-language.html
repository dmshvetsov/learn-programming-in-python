No we have understanding what data is. How we can store it in a program language such as Python?

We can store data in computer memory.

To do so we can use a line with an instruction like bellow:


						a_string_variable = "this is a python string"
						a_year_variable = 2020

Here on the first line we store string data and on the line 2 store a number data.

We store this data in a computer memory and need something to refer to it.

On the left side of = sign we have a name:


						a_string_variable = "this is a python string"
						a_year_variable = 2020

a_string_variable it is a name that points to the string in the memory and a_year_variable it is a name that points to the number in the memory.

Variables

Name that points into a piece of data in the computer memory called variables.

Often in programming thing are called after a meaning.

Variable are good example of it. It is called this way because they are used to store variable data and can change over time in Python.

Here variable piece_of_data store 4 different values:


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0

On each line we assign different data to a variable.

First, two stings on line 1 and 2.

Then, two numbers on line 2 and 3.


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0

On each line, the state of the computer's memory and as a result of our 4 lines program changes.

On line 1 our program stores string "abra"


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra"]
 ^
 piece_of_data

[] characters represents the computer memory and what inside of it.

On line 2 our program stores string "cadabra"


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra", "cadabra"]
          ^
          piece_of_data

On line 3 our program stores number 37


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra", "cadabra", 37]
                    ^
                    piece_of_data

On line 4 our program stores number 0


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra", "cadabra", 37, 0]
                        ^
                        piece_of_data

At the end of our 4 line program we in the memory have all 4 data that we created and piece_of_data variable points to the last data that we created – number 0


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra", "cadabra", 37, 0]
                        ^
                        piece_of_data

At the end of the program we have access to 0. This means that we can refer only to the data that we stored in a named variable.

If we add another line with another 0 at line 5 to our program this will create another 0 to the memory:


						piece_of_data = "abra"
						piece_of_data = "cadabra"
						piece_of_data = 37
						piece_of_data = 0
						piece_of_data = 0
How variable pointing (or referring) to data in the memory: 

["abra", "cadabra", 37, 0, 0]
                           ^
                           piece_of_data
